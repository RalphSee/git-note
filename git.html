本地仓库内容提交到远程仓库

远程仓库地址 ：https://github.com/zfpx/201804JS.git
① 关联本地仓库和远程仓库
git remote add zfpx(远程仓库名) https://github.com/zfpx/201804JS.git
git remote -v 查看关联的远程仓库
git remote rm zfpx 删除关联的远程仓库

② 推送本地仓库内容到远程仓库
git push zfpx master 推送到远程仓库的主分支

  1. 进入文件夹 打开git命令窗口 -> git init 初始化本地git仓库
  2. git add . 提交到暂存区
  3. git commit -m “提交的信息”
  4. 去github新建仓库 点+号->new respository->create respository
    会生成一个远程仓库地址 https://github.com/zfpx/201804JS.git
  5. 进入本地仓库文件夹 关联本地仓库和远程仓库
    git remote add origin https://github.com/zfpx/201804JS.git
  6. git push origin master 把本地仓库内容推送到远程仓库

另一种提交本地项目的方式

  • git clone https://github.com/RalphSee/test.git
    相当于做了三件事
    1、建了一个本地git仓库，仓库名和远程仓库名一样
    2、本地仓库和远程仓库关联起来了
    3、远程仓库的内容已经拉去到本地仓库
  • git add .
  • git commit “”
  • git push origin master
    远程仓库内容和本地仓库内容不一样时，需要两个仓库保持同步后，才能往远程仓库推送内容，同步的做法是git pull origin master 把远程仓库内容先拉去到本地仓库

暂存区

第一次提交时git add . 工作区->暂存区
把提交到暂存区的内容删除 git rm –cached 文件名
若工作区内容修改了，暂存区就删除不了，只能通过 git add .再提交暂存区

查看工作区和暂存区的区别
git diff

历史区

git commit -m “” 提交到历史区 生成一个版本

查看工作区，暂存区，历史区代码的区别

git diff a.txt 查看工作区和暂存区的区别
git diff master 工作区和历史区
git diff –cached 暂存区和历史区

查看提交历史

git log 查看详细
git log –oneline 简洁的一行日志

撤销

把提交的版本覆盖工作区的内容（恢复历史版本）
git reset HEAD 文件名 选择历史区的版本恢复到暂存区
git checkout 文件名 把暂存区内容恢复到工作区

git commit -am ” 一次性把工作区的代码提交到历史区

通过版本号恢复到之前提交的版本

git reset –hard 版本号 切换到之前的版本
git log –oneline 只能看到之前的版本
git reflog 查看所有的版本
git reset –hard HEAD^ 回到指定版本的上一个版本

  • 代码git commit -m”” 提交到历史区 这个代码就属于这个分支

    • git stash apply stash@{0} 指定过渡区的内容恢复到工作区

如何把项目发布到github,并生成在线地址 （只能放静态资源）

  • git branch gh-pages 并切换到gh-pages
  • 把项目提交到这个分支 git commit -m”“
  • 提交到远程仓库 git pull origin gh-pages
